HTTP协议

发表于 2019-01-17 | 阅读次数:
字数统计: 1.8k | 阅读时长 ≈ 6

HTTP简介

  1. HTTP协议是超文本传输协议，用于万维网(www)服务器传输超文本到本地浏览器的协议。
  2. URL：统一资源定位符。
  3. URI: 统一资源标识符。
  4. HTTP是一个属于应用层的面向对象的协议，由于其简捷、快速的方式，适用于分布式超媒体信息系统。它于1990年提出，经过几年的使用与发展，得到不断地完善和扩展。目前在WWW中使用的是HTTP/1.0的第六版，HTTP/1.1的规范化工作正在进行之中，而且HTTP-NG(Next Generation of HTTP)的建议已经提出。

    主要特点

  5. 支持B/S以及C/S模式。
  6. 简单快速： 客户向服务器请求服务时，只需传送请求方法和路径。请求方法常用的有GET、HEAD、POST。每种方法规定了客户与服务器联系的类型不同。由于HTTP协议简单，使得HTTP服务器的程序规模小，因而通信速度很快。
  7. 灵活： HTTP允许传输任意类型的数据对象。正在传输的类型由Content-Type加以标记。
  8. 无连接： 无连接的含义是限制每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。采用这种方式可以节省传输时间。
  9. 无状态： HTTP协议是无状态协议。无状态是指协议对于事务处理没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，则它必须重传，这样可能导致每次连接传送的数据量增大。另一方面，在服务器不需要先前信息时它的应答就较快。

    HTTP之URL

    URI：统一资源标识符，用来唯一的标识一个资源。

URL：统一资源定位符。

URI和URL的关系

URI用字符串标识某一互联网资源，而URL表示资源的地点，即互联网所处的位置，所以URL是URI的子集。

HTTP协议之请求消息Request

http请求由请求行、请求头部、请求正文 组成。(注意：请求头部和请求正文用换行符隔开)

请求行

请求行是请求方法开头，以空格隔开，后面跟着请求URI和协议版本。

  • 请求方法
  • GET 获取资源
  • POST 向服务器端发送数据，传输实体主体
  • PUT 传输文件
  • HEAD 获取报文首部
  • DELETE 删除文件
  • OPTIONS 询问支持的方法
  • TRACE 追踪路径

    Get请求举例

  • get表示一个读取请求，/ 表示URL的路径，HTTP/1.1表示采用的HTTP版本是1.1。
  • Host表示请求的域名。

第一部分：请求行，用来说明请求类型,要访问的资源以及所使用的HTTP版本.

第二部分：请求头部，紧接着请求行（即第一行）之后的部分，用来说明服务器要使用的附加信息
从第二行起为请求头部，HOST将指出请求的目的地.User-Agent,服务器端和客户端脚本都能访问它,它是浏览器类型检测逻辑的重要基础.该信息由你的浏览器来定义,并且在每个请求中自动发送等等

第三部分：请求数据也叫主体，可以添加任意的其他数据。

HTTP协议之响应消息Response

HTTP响应也由三个部分组成，分别是：状态行、消息报头和响应正文。

状态行

  • 200 OK 表示一个成功的响应。OK表示说明。失败的响应有404 Not Found。

    消息报头

  • Content-Type表示响应的内容，浏览器根据Content-Type来判断响应的内容是网页还是图片，是视频还是音乐。本例中text/html是网页的意思。

    响应正文

    HTTP响应的Boby就是HTML源码。

    介绍一下响应的状态码

类别 原因
1XX 指示信息–表示请求已接收，继续处理
2XX 成功–表示请求已被成功接收、理解、接受
3XX 重定向–要完成请求必须进行更进一步的操作
4XX 客户端错误–请求有语法错误或请求无法实现
5XX 服务器端错误–服务器未能实现合法的请求

常见的状态码

状态码 状态码的含义
200 OK 客户端请求成功
400 Bad Request 客户端请求有语法错误，不能被服务器所理解
401 Unauthorized 请求未经授权，这个状态代码必须和WWW-Authenticate报头域一起使用
403 Forbidden 服务器收到请求，但是拒绝提供服务
404 Not Found 请求资源不存在，eg：输入了错误的URL
500 Internal Server Error 服务器发生不可预期的错误
503 Server Unavailable 服务器当前不能处理客户端的请求，一段时间后可能恢复正常

HTTP的请求流程

第一步：浏览器向服务器发送HTTP请求。(通过TCP套接字连接)

  • 请求包括：
  • 方法 GET还是POST，Get仅请求资源，POST会附带用户数据。
  • 路径
  • 域名
  • 以及其他相关的Header

第二步：服务器向浏览器返回HTTP响应。

  • 响应包括：
  • 响应状态码(详情请看上面状态码)
  • 响应类型(由Content-Type决定)
  • 其他相关的Header
  • 通常HTTP响应会携带内容，也就是有Boby，网页的HTML源码也就在Boby中。

第三步： 如果浏览器还需要向服务器请求其他资源，比如图片，就再次发送HTTP请求，重复第一步和第二步。

Web采用的HTTP协议采用了非常简单的请求-响应模式，从而大大简化了开发。当我们编写一个页面时，我们只需要在HTTP请求中把HTML发送出去，不需要考虑如何附带图片、视频等，浏览器如果需要请求图片和视频，它会发送另一个HTTP请求，因此，一个HTTP请求只处理一个资源。

GET和Post请求的区别

  • GET：从指定的资源请求数据。
  • POST: 向指定的资源提交要被处理的数据。

  • 类型|GET | POST
    —|—|—
    后退/刷新|不会产生害处 |数据会被重新提交
    书签| 可被存为书签 | 不可存为书签
    缓存|可以 | 不可以
    编码类型|application/x-www-form-urlencoded | pplication/x-www-form-urlencoded 或 multipart/form-data。为二进制数据使用多重编码。
    历史记录| 参数可保存在历史记录中| 不能保存
    对数据长度的限制| 当发送数据时，get方法向URL添加数据，URL的长度是受限制的 | 无限制
    对数据类型的限制| 只允许ASCII字符 |没有限制，也允许二进制数据。
    安全性| 安全性较差，因为数据是URL的一部分 |POST比get更安全，因为参数存在于Boby中，且不会被保存到浏览器历史或WEB日志中。
    可见性|数据在URL中对所有人都是可见的 | 数据不会显示在URL中。
